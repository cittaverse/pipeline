# 临床数据收集协议设计

**版本**: v1.0  
**日期**: 2026-03-16  
**依据**: RT Meta-analysis (PMC, Jan 2026) — MCI 亚组仅基于 3 项小样本研究  
**目标**: 在产品中内置结构化数据收集机制，填补 MCI 证据空白

---

## 🎯 设计目标

### 核心问题

> **Meta 分析发现**: 回忆疗法对 MCI 亚组的疗效证据仅基于 3 项研究，样本量小，需谨慎解读。

**产品机会**:
- 通过产品化规模收集真实世界数据
- 建立 MCI 人群专属的叙事疗法证据库
- 为未来学术论文/临床试验提供基线数据

### 数据收集原则

| 原则 | 说明 | 实现方式 |
|------|------|---------|
| **用户同意优先** | 所有数据收集需明确告知并获得同意 | 知情同意书 + 单独授权选项 |
| **最小必要** | 仅收集研究必需的数据 | 数据字典明确定义每个字段用途 |
| **去标识化** | 研究数据与个人身份信息分离 | 双数据库设计（身份库 + 数据库） |
| **可撤回** | 用户可随时撤回同意并删除数据 | 数据删除 API + 审计日志 |

## 🔐 数据架构

### 数据库设计

```
┌─────────────────────────────────────────────────────────────┐
│                     身份数据库 (PII)                        │
│  (独立部署，严格访问控制，与数据库物理隔离)                   │
├─────────────────────────────────────────────────────────────┤
│ participant_id (UUID) ←─── 主键                             │
│ phone_hash (HASH)      ←─── 脱敏手机号                       │
│ name_encrypted (AES)   ←─── 加密姓名                         │
│ emergency_contact      ←─── 紧急联系人                       │
│ consent_record         ←─── 同意书记录 (时间/IP/版本)          │
│ withdrawal_status      ←─── 撤回状态                         │
└─────────────────────────────────────────────────────────────┘
                            │
                            │ participant_id (外键关联)
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                     研究数据库 (De-identified)               │
│  (可导出用于分析，不含个人身份信息)                           │
├─────────────────────────────────────────────────────────────┤
│ participant_id (UUID) ←─── 主键                             │
│ demographics          ←─── 人口学信息表                      │
│ intervention_data     ←─── 干预过程表                        │
│ clinical_outcomes     ←─── 临床评估表                        │
│ narrative_metrics     ←─── 叙事质量表                        │
│ adverse_events        ←─── 不良事件表                        │
└─────────────────────────────────────────────────────────────┘
```

### 数据流向

```
用户口述 → ASR 转录 → 叙事评分 → 数据存储
   │                       │
   ▼                       ▼
知情同意校验           质量校验 (范围/完整性)
   │                       │
   ▼                       ▼
同意？───否──→ 仅服务，不存储研究数据
   │
   是
   │
   ▼
去标识化 → 研究数据库
   │
   ▼
定期导出 (周/月) → 分析 Pipeline
```

## 📋 知情同意书模板

### 核心条款

```markdown
# 知情同意书 — CittaVerse 叙事疗法研究

## 研究目的
本研究旨在评估 AI 辅助叙事疗法对轻度认知障碍 (MCI) 人群的认知功能、
情绪状态和生活质量的影响。研究数据将用于学术发表和产品改进。

## 数据收集范围
我们收集以下类型的数据：
1. 基本信息：年龄、性别、受教育年限、MoCA 筛查分数
2. 干预数据：会话录音（经 ASR 转文字）、会话时长、参与度评分
3. 临床评估：MoCA、GDS-15、QOL-AD、SWLS（基线、第 8 周、随访）
4. 叙事质量：AI 自动评分的 6 个维度指标

## 数据使用方式
1. 去标识化后用于统计分析
2. 匿名引用用于学术论文发表
3. 聚合数据用于产品改进
4. **不会**将个人身份信息出售给第三方

## 您的权利
1. 随时撤回同意，无需理由
2. 撤回后，您的个人身份信息将被删除
3. 撤回前已去标识化的数据可能保留用于已完成分析
4. 可随时要求查看您的个人数据

## 数据保护
1. 身份信息与研究数据物理隔离存储
2. 数据传输使用 HTTPS 加密
3. 仅授权研究人员可访问去标识化数据
4. 数据保存期限：研究结束后 5 年

## 联系方式
研究负责人：[姓名]
伦理委员会：[机构名称 + 联系方式]
```

### 同意流程

```
1. 用户阅读知情同意书 (强制停留≥60 秒)
   ↓
2. 用户勾选同意选项
   ↓
3. 系统记录同意时间、IP、同意书版本
   ↓
4. 生成 participant_id (UUID)
   ↓
5. 身份信息存入 PII 数据库
   ↓
6. 研究数据存入去标识化数据库
```

## 📈 分析 Pipeline

### 主要分析计划

#### 1. 主要终点分析

**假设**: 8 周干预后，MCI 组 MoCA 分数显著改善

```python
# 统计分析计划
primary_analysis = {
    "outcome": "moca_week8 - moca_baseline",
    "test": "paired t-test (正态) / Wilcoxon signed-rank (非正态)",
    "alpha": 0.05,
    "effect_size": "Cohen's d",
    "subgroup": ["MCI (MoCA 18-25)", "正常 (MoCA 26-30)"]
}
```

#### 2. 剂量 - 效应分析

**假设**: 会话次数与认知改善呈正相关

```python
dose_response = {
    "predictor": "session_number (1-16)",
    "outcome": "moca_change",
    "test": "linear regression",
    "covariates": ["age", "education", "baseline_moca"]
}
```

#### 3. 叙事质量 - 认知改善相关性

**假设**: 叙事评分改善与 MoCA 改善正相关

```python
narrative_cognition = {
    "predictor": "narrative_score_change (week8 - baseline)",
    "outcome": "moca_change",
    "test": "Pearson/Spearman correlation",
    "exploratory": ["各维度评分与认知域的相关性"]
}
```

#### 4. 亚组分析

| 亚组 | 分层变量 | 分析目的 |
|------|---------|---------|

| 性别 | 男 / 女 | 性别差异 |

## 🔒 数据安全与合规

### 合规框架

| 法规 | 适用性 | 合规措施 |
|------|--------|---------|
| 《个人信息保护法》(PIPL) | ✅ 适用 | 单独同意、最小必要、可撤回 |
| 《人类遗传资源管理条例》 | ⚠️ 部分适用 | 不涉及基因数据，无需审批 |
| 《涉及人的生物医学研究伦理审查办法》 | ✅ 适用 | 伦理委员会审批 + 知情同意 |
| GDPR (如涉欧盟用户) | ⚠️ 可能适用 | 数据跨境传输评估 |

### 安全措施

| 层级 | 措施 |
|------|------|
| 传输层 | HTTPS/TLS 1.3 加密 |
| 存储层 | AES-256 加密 (身份信息) |
| 访问层 | RBAC 权限控制 + 双因素认证 |
| 审计层 | 所有数据访问记录日志 |
| 备份层 | 每日增量备份 + 每周全量备份 |
