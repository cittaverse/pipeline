## Overview

This guide provides step-by-step instructions for integrating LLM-enhanced feature extraction into the narrative scoring pipeline. The v0.7 architecture uses a **hybrid approach**: rule-based scoring as the primary engine, augmented by LLM-based implicit feature detection.

### Key Principles

1. **Graceful Degradation**: If LLM API fails, falls back to rule-only mode (v0.6 behavior)
2. **Cost Control**: LLM calls are optimized for minimal token usage (~¥0.0028/narrative)
3. **Explainability**: LLM outputs are structured (JSON) and auditable
4. **Modularity**: LLM integration is optional and configurable

---

## Architecture

```
Input Text
    │
    ├─── Rule-Based Feature Extractor ──→ Explicit features
    │        ├─ Explicit emotion words (90-word lexicon)
    │        ├─ Heuristic event boundaries (temporal/spatial markers)
    │        └─ Explicit causal markers (lexical cues)
    │
    ├─── LLM Feature Extractor ──────────→ Implicit features
    │        ├─ Implicit emotions (e.g., "心里空落落的" → sadness)
    │        ├─ Semantic event boundaries (beyond heuristics)
    │        └─ Implicit causal links (inferred from context)
    │
    └─── Fusion Layer ────────────────────────────┘
                     │
                     ↓
              Rule-Based Scorer (enhanced features)
                     │
                     ↓
              Dimension Scores (0-5 scale)
```

## Step-by-Step Integration

### Step 1: Configuration

Copy the example config and customize:
```bash
cp config/llm_config.example.yaml config/llm_config.yaml
```

Edit `config/llm_config.yaml`:
```yaml
api_key: null  # Uses DASHSCOPE_API_KEY env var if null
model: qwen-plus
timeout: 5
max_retries: 2
fallback_to_rule_only: true
```

### Step 2: Import LLM Feature Extractor

In your scoring script (e.g., `score_narrative.py`):
```python
from src.llm_feature_extractor import LLMFeatureExtractor, LLMConfig

# Load configuration
config = LLMConfig.from_yaml('config/llm_config.yaml')
# Or use environment variables:
# config = LLMConfig()  # Uses DASHSCOPE_API_KEY from env

# Initialize extractor
extractor = LLMFeatureExtractor(config)
```

### Step 3: Extract LLM Features

```python
text = "那年夏天，我考上了大学。收到录取通知书的那天，全家人都很高兴。"

# Extract LLM features
llm_features = extractor.extract(text)

# Access results
print(f"Detected {len(llm_features.emotions)} emotions")
print(f"Detected {len(llm_features.events)} event boundaries")
print(f"Detected {len(llm_features.causal_relations)} causal relations")
print(f"LLM API success: {llm_features.llm_call_success}")
print(f"Used fallback: {llm_features.used_fallback}")
```

### Step 4: Integrate with Rule-Based Scorer

**Option A: Manual Fusion** (recommended for fine-grained control):
```python
from src.scorer import score_narrative

# Get rule-based features
rule_features = score_narrative(text, return_features=True)

# Get LLM features
llm_features = extractor.extract(text)

# Fuse features
if not llm_features.used_fallback:
    # Enhance emotion count with implicit emotions (weighted at 0.5)
    implicit_emotions = [e for e in llm_features.emotions if e.emotion_type == 'implicit']
    enhanced_emotion_count = rule_features['emotion_count'] + len(implicit_emotions) * 0.5
    
    # Enhance causal count
    enhanced_causal_count = rule_features['causal_count'] + len(llm_features.causal_relations)
    
    # Use enhanced features in scoring
    final_scores = compute_scores(
        emotion_count=enhanced_emotion_count,
        causal_count=enhanced_causal_count,
        event_boundaries=llm_features.events,  # Use LLM event boundaries
    )
else:
    # Fallback to rule-only scoring
    final_scores = compute_scores(**rule_features)
```

**Option B: Automatic Fusion** (simpler, less control):
```python
from src.llm_feature_extractor import LLMFeatureExtractor

extractor = LLMFeatureExtractor(config)

# Rule-based features
rule_features = score_narrative(text, return_features=True)

# Automatic fusion with fallback
enhanced_features = extractor.extract_with_fallback(text, rule_features)

# Use enhanced features
final_scores = compute_scores(**enhanced_features)
```

### Step 5: Error Handling

```python
try:
    llm_features = extractor.extract(text)
    
    if llm_features.used_fallback:
        logger.warning("LLM extraction failed, using rule-only fallback")
    
    if llm_features.total_cost > 0:
        logger.info(f"LLM extraction cost: ¥{llm_features.total_cost:.4f}")
        
except Exception as e:
    logger.error(f"LLM extraction failed: {e}")
    # Handle gracefully (e.g., use rule-only scoring)
```

## Cost Estimation

### Per-Narrative Cost

**DashScope qwen-plus pricing** (as of 2026-03):
- Input: ¥0.0028 per 1K tokens
- Output: ¥0.0028 per 1K tokens

**Typical narrative** (200 Chinese characters):
- Input tokens: ~100 (prompt + text)
- Output tokens: ~200 (JSON response)
- **Cost per narrative**: ~¥0.00084

**Pilot RCT** (N=50 participants, 3 narratives each = 150 narratives):
- **Total cost**: ~¥0.13

**Large-scale study** (N=500, 3 narratives each = 1500 narratives):
- **Total cost**: ~¥1.26

### Cost Optimization Strategies

1. **Use qwen-turbo** for development/testing (cheaper, slightly less capable)
2. **Batch processing** for large datasets (reduce API call overhead)
3. **Caching** for repeated narratives (use `use_cache: true` in config)
4. **Selective LLM usage** (e.g., only for narratives with low rule-based confidence)

## Troubleshooting

### Issue: "DASHSCOPE_API_KEY not set"

**Solution**: Set environment variable:
```bash
export DASHSCOPE_API_KEY="your-api-key-here"
```

Or update `config/llm_config.yaml`:
```yaml
api_key: "your-api-key-here"
```

### Issue: "DashScope not available"

**Solution**: Install DashScope SDK:
```bash
pip install dashscope
```

### Issue: "LLM API call failed after all retries"

**Possible causes**:
1. Network connectivity issues
2. API rate limit exceeded
3. Invalid API key
4. DashScope service outage

**Solutions**:
- Check network connection
- Reduce request rate (increase `rate_limit_delay`)
- Verify API key is valid
- Check DashScope status: https://status.aliyun.com/

### Issue: "Failed to parse LLM JSON response"

**Possible causes**:
1. LLM output format deviation
2. Prompt template issue
3. Model version change

**Solutions**:
- Check raw LLM response in debug logs
- Update prompt template for stricter JSON enforcement
- Try different model (e.g., `qwen-max` for better instruction following)

### Issue: High fallback rate (>10%)

**Investigation**:
```python
features = extractor.extract(text)
print(f"LLM calls made: {features.llm_calls_made}")
print(f"LLM calls failed: {features.llm_calls_failed}")
print(f"Fallback used: {features.used_fallback}")
```

**Solutions**:
- Increase timeout (e.g., `timeout: 10`)
- Increase retries (e.g., `max_retries: 3`)
- Check API key permissions and rate limits

## Migration from v0.6 to v0.7

### Backward Compatibility

v0.7 is **fully backward compatible** with v0.6:
- All v0.6 tests pass unchanged
- Rule-based scoring logic is unchanged
- LLM features are additive (opt-in)

### Upgrade Steps

1. **Update repository**:
   ```bash
   git pull origin main
   pip install -r requirements.txt
   ```

2. **Add configuration**:
   ```bash
   cp config/llm_config.example.yaml config/llm_config.yaml
   ```

3. **Set API key**:
   ```bash
   export DASHSCOPE_API_KEY="your-api-key-here"
   ```

4. **Test with fallback** (safe mode):
   ```python
   config = LLMConfig(fallback_to_rule_only=True)
   extractor = LLMFeatureExtractor(config)
   ```

5. **Enable LLM features**:
   ```python
   config = LLMConfig(
       use_emotion_detection=True,
       use_event_segmentation=True,
       use_causality_detection=True,
   )
   ```

6. **Run benchmark tests**:
   ```bash
   pytest tests/test_benchmark.py -v
   ```

7. **Deploy to production** (after validation)
