# Pipeline 架构说明

> 极简设计 · 模块化 · 可解释

---

## 文件结构

```
pipeline/
├── PROGRAM.md          # 评估协议 (Agent 指令)
├── ARCHITECTURE.md     # 本文件
├── prepare.py          # 固定：数据准备 + 工具函数 (不修改)
├── assess.py           # 可变：评估逻辑 (可配置)
├── requirements.txt    # 依赖
└── examples/           # 示例输入输出
```

---

## 核心设计原则

### 1. 极简主义

受 Karpathy auto-research 启发：
- **少文件**：核心逻辑集中在 1-2 个文件
- **少依赖**：仅必要的第三方库
- **少配置**：默认配置即可运行

### 2. 明确边界

| 文件 | 用途 | 是否可修改 |
|------|------|------------|
| prepare.py | 工具函数、常量定义 | ❌ 固定 |
| assess.py | 评估逻辑、LLM 调用 | ✅ 可配置 |
| PROGRAM.md | Agent 指令 | ✅ 可迭代 |

### 3. 固定预算

评估时间上限：**30 秒/文本** (1000 字以内)

超时处理：
- 返回部分结果
- 标注"评估不完整"

### 4. 单一主指标

**overall_score** (0-100) 越高越好

子维度用于诊断，不用于横向比较。

## 神经符号架构

```
┌─────────────────────────────────────────┐
│         神经层 (LLM)                     │
│  - 语义理解                              │
│  - 细节分类                              │
│  - 事件边界检测                          │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│         符号层 (图论)                     │
│  - NetworkX 事件图构建                    │
│  - 连通性分析                            │
│  - 最长路径 (叙事主线)                    │
│  - 标准化计分                            │
└─────────────────────────────────────────┘
```

**为什么混合**：
- 纯 LLM：黑箱、不可解释
- 纯规则：僵化、体验差
- 混合：LLM 理解语义 + 图论保证可解释

## 评估指标

### 核心指标

| 指标 | 范围 | 解读 |
|------|------|------|
| overall_score | 0-100 | 综合叙事质量 |
| internal_details_score | 0-100 | 个人感官记忆丰富度 |
| external_details_score | 0-100 | 历史社会背景描述 |
| coherence_score | 0-100 | 叙事连贯性 |

### 辅助指标

| 指标 | 类型 | 用途 |
|------|------|------|
| word_count | 整数 | 文本长度控制 |
| event_count | 整数 | 事件数量 |
| processing_time_ms | 整数 | 性能监控 |
| confidence | 0-1 | 评估置信度 |

## 扩展点

### 可配置项

```python
# assess.py 顶部配置
CONFIG = {
    "model": "qwen-plus",      # LLM 模型
    "language": "zh-CN",        # 语言
    "max_words": 1000,          # 最大字数
    "timeout_seconds": 30,      # 超时时间
    "api_base": None,           # 自定义 API 端点
}
```

### 可扩展模块

1. **新的细节类型**：在 PROGRAM.md 中定义
2. **新的计分算法**：修改 assess.py 中的 ScoreCalculator
3. **新的输出格式**：修改 ReportGenerator

## 故障排除

### 常见问题

**Q: LLM API 调用失败**
- 检查 API Key 是否设置
- 检查网络连接
- 查看 API 配额

**Q: 评估结果异常低**
- 检查文本是否为有效叙事
- 检查语言是否匹配 (中文/英文)
- 查看 evidence 字段了解依据

**Q: 运行超时**
- 文本过长，建议分段
- LLM 响应慢，切换模型
- 网络问题，检查连接
